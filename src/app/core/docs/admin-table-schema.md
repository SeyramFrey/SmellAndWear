# Admin Table Schema

This document describes the actual structure of the `admin` table in Supabase for the Smell&Wear project.

## Table: `admin`

### Actual Columns (Current Implementation)

| Column Name | Type      | Constraints                    | Description                     |
|-------------|-----------|--------------------------------|---------------------------------|
| `id`        | BIGINT    | PRIMARY KEY, IDENTITY          | Auto-increment admin ID        |
| `username`  | TEXT      | NULLABLE                       | Admin username (for login)     |
| `password`  | TEXT      | NULLABLE                       | Plain text password (demo only) |

### Current Database Structure

```sql
-- Existing admin table structure
CREATE TABLE admin (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username TEXT,
    password TEXT
);
```

### Security Notes

⚠️ **Important Security Considerations:**

1. **Password Storage**: The current implementation uses plain text passwords for demo purposes. In production, implement proper password hashing with bcrypt or similar.

2. **RLS Enabled**: The admin table has Row Level Security (RLS) enabled in Supabase with a policy allowing public read access for authentication.

3. **Authentication**: The system currently authenticates against the existing admin user "Hans".

4. **RLS Policy**: A public read policy has been created to allow authentication: `"Allow public read for authentication"`

### Troubleshooting

**Common Issues:**

- **406 Not Acceptable / PGRST116 Error**: This indicates RLS is blocking access. Ensure the RLS policy exists:
  ```sql
  CREATE POLICY "Allow public read for authentication" ON admin FOR SELECT USING (true);
  ```

- **No rows returned**: Verify the username exists in the database and matches exactly (case-sensitive).

- **Connection issues**: 
  - Ensure the admin table exists in your Supabase database
  - Verify Supabase connection credentials in environment files
  - Check browser console for detailed error messages

**Testing Steps:**
1. Test direct database access via Supabase dashboard
2. Verify RLS policies are correctly configured
3. Check network tab in browser dev tools for API responses
4. Confirm credentials: Username: `Hans`, Password: `Admin@1234`
