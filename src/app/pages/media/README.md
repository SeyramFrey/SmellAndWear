# Media Management Feature

This directory contains the complete media management implementation for the SmellAndWear Angular application, integrated with Supabase for storage and database operations.

## Architecture

### Components
- **MediasComponent** (`medias/medias.component.ts`): Main component for displaying and managing media files
- **MediaModel** (`medias/media.model.ts`): TypeScript interface aligned with Supabase schema

### Services
- **MediaService** (`../../../core/services/media.service.ts`): Production-ready service handling all Supabase operations

## Database Schema

The implementation uses the existing Supabase `media` table:

```sql
CREATE TABLE media (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  media_path VARCHAR NULL,
  media_name TEXT NULL
);
```

## Storage

Media files are stored in Supabase Storage buckets following the same approach as variants-list:
- **medias**: Primary storage bucket for uploaded files under `public_images/` folder
- **public-images**: Alternative bucket for public images (variants use `variants/` folder)

### File Organization
```
medias bucket:
├── public_images/
│   ├── media_1234567890_image1.jpg
│   ├── media_1234567891_document.pdf
│   └── media_1234567892_video.mp4
```

## Security (RLS Policies)

### Media Table Policies
- **Read**: Public access for all users (suitable for e-commerce product images)
- **Insert/Update/Delete**: Authenticated users only

### Storage Policies
- **Read**: Public access for viewing files
- **Insert/Update/Delete**: Authenticated users only

## Usage Examples

### Basic Component Usage
```typescript
import { MediasComponent } from './pages/media/medias/medias.component';

// The component is standalone and can be used directly
<app-medias></app-medias>
```

### Using MediaService
```typescript
import { MediaService } from './core/services/media.service';

constructor(private mediaService: MediaService) {}

// Get all media files
this.mediaService.getMedias(0, 20).subscribe(result => {
  if (!result.error) {
    console.log('Media files:', result.data);
    console.log('Total count:', result.total);
  }
});

// Upload a file
this.mediaService.uploadMedia(file).subscribe(result => {
  if (!result.error) {
    console.log('Upload successful:', result.data);
  }
});

// Delete a media file
this.mediaService.deleteMedia(mediaId).subscribe(success => {
  if (success) {
    console.log('Delete successful');
  }
});

// Get public URL for a file
const url = this.mediaService.getMediaUrl('path/to/file.jpg');
```

## File Types Supported

The service automatically detects and categorizes files:

- **Images**: jpg, jpeg, png, gif, webp, svg
- **Videos**: mp4, avi, mov, wmv, webm
- **Audio**: mp3, wav, flac, ogg
- **Documents**: pdf, doc, docx
- **Archives**: zip, rar, 7z
- **Other**: All other file types

## Configuration

### Environment Setup
Ensure your environment files have the correct Supabase configuration:

```typescript
// src/environments/environment.ts
export const environment = {
  supabase: {
    url: 'YOUR_SUPABASE_URL',
    key: 'YOUR_SUPABASE_ANON_KEY'
  }
};
```

### Storage Bucket Setup
The service uses the 'medias' bucket by default. Ensure it exists and is properly configured in your Supabase project.

## Error Handling

The service includes comprehensive error handling:
- Network errors are caught and logged
- Failed uploads clean up partial storage uploads
- RLS policy violations are handled gracefully
- Invalid file operations return meaningful error messages

## Performance Features

- **Pagination**: Configurable page sizes (default: 20 items)
- **Search Debouncing**: 300ms delay to prevent excessive API calls
- **Lazy Loading**: Images are loaded only when visible
- **Optimistic Updates**: UI updates immediately for better UX

## Security Considerations

- All write operations require authentication
- RLS policies protect against unauthorized access
- File uploads are validated on the client side
- Storage policies prevent unauthorized file access
